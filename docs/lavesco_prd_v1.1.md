# **3\. 화면 구조 및 UX 플로우**

## **3-1. 진입/공통**

* 스플래시 — 앱 아이콘 1.5초 후 로그인 여부 분기

* 로그인 / 회원가입

  * 이메일 \+ 소셜(Google · Apple) 로그인

  * Apple 로그인은 App Store 제출 필수 (규정)

  * 약관 / 개인정보 동의 (필수 / 선택 분리 표시)

* 권한 요청 (선택적 — 필요 시점에 요청)

  * 알림(FCM): 초대 수신, 라운드 시작·종료 알림

  * 연락처: 친구 초대 기능 사용 시에만 요청

💡 권한 요청은 앱 진입 즉시가 아닌, 해당 기능을 처음 사용할 때 컨텍스트와 함께 요청 (iOS/Android 정책 준수)

## **3-2. 탭 구조 (4탭)**

| 탭 | 레이블 | 아이콘 예시 | 주요 역할 |
| :---- | :---- | :---- | :---- |
| 1 | 홈 | 🏠 | 대시보드 · 빠른 시작 · 초대 코드 입력 |
| 2 | 라운드 | ⛳ | 라운드 생성/참가/진행 · 스코어 입력 · 보드 |
| 3 | 코스 | 🗺️ | 코스 검색 · 홀 정보 · 즐겨찾기 |
| 4 | 내 기록 | 👤 | 라운드 히스토리 · 통계 · 프로필/설정 |

## **3-3. 홈 탭 (A)**

### **A-1. 홈 대시보드**

* "새 라운드 만들기" CTA

* "초대 코드로 참가" CTA

* 진행중 라운드가 있을 경우 상단에 배너 표시 (빠른 복귀)

* 최근 라운드 카드 (진행중 / 완료)

* 즐겨찾는 코스 바로가기 (선택, 2차)

💡 진행중 라운드가 1개 이상이면 홈 진입 시 "라운드 계속하기" 바텀시트를 1회 노출 (UX 개선)

### **A-2. 초대 코드 입력 (Join by Code)**

* 6자리 영숫자 코드 입력 (자동 대문자)

* 참가자 닉네임 미리보기 → 참가 버튼

* 만료/비활성 코드 인입 시 에러 메시지

### **A-3. 딥링크 처리 화면**

* 앱 미설치 시: 스토어로 유도 \+ defer deep link 처리

* 앱 설치 후: 라운드 정보 미리보기 → 참가

* 이미 참가한 라운드라면 바로 라운드 상세로 이동

## **3-4. 라운드 탭 (B) — 핵심**

### **B-1. 라운드 목록**

* 상태 필터: 진행중 / 예정 / 완료

* 범위 필터: 내 라운드 / 초대받은 라운드

* 완료 라운드 무한 스크롤 (20건씩 페이지네이션)

* 빈 상태(Empty State) 처리: 필터별 적절한 안내 문구

### **B-2. 라운드 생성 플로우**

* Step 1 — 골프장 선택

  * 코스 검색 (이름 / 지역)

  * 즐겨찾기 코스 최상단 노출

  * 검색 결과 없음 화면 \+ 코스 요청 CTA (2차)

* Step 2 — 라운드 설정

  * 날짜 / 시간 선택 (DatePicker)

  * 전반 / 후반 코스 선택 (27홀 코스 대응)

  * 티셋 선택 (White / Blue / Red 등)

  * 라운드 이름 / 메모 (선택)

* Step 3 — 멤버 초대

  * 초대 코드 표시 \+ 복사 / 공유 버튼

  * 친구 목록에서 직접 초대 (선택, 2차)

* Step 4 — 생성 완료

  * "라운드 시작하기" / "나중에 입력하기" 선택

⚠️ Step 이동 시 뒤로가기 안내: "라운드 초안이 저장됩니다" 알림 (DRAFT 상태 자동 저장)

### **B-3. 라운드 상세 (진행 허브)**

* 코스 / 티셋 / 날짜 요약

* 참가자 목록 \+ 상태 (참가 대기 / 참가 / 입력 진행)

* 현재 총타 / 스코어보드 미리보기 (상위 3명)

* 호스트 전용 액션: 초대 관리, 라운드 종료

* 공통 액션: 스코어 입력, 스코어보드 전체 보기

### **B-4. 스코어 입력**

* 홀 번호 / Par / 거리 표시

* 내 타수 \+/- 빠른 입력 (탭 \+ 드래그 모두 지원)

* 퍼트수 \+/- (선택 입력)

* 세부 기록 (선택): Fairway / OB / 헤저드 / 벙커

* 이전 / 다음 홀 스와이프 내비게이션

* 홀 목록 그리드 (1\~18 전체 타수 한눈에 조회, 홀 클릭 → 상세 입력)

💡 오프라인 상태에서도 입력 가능 (로컬 캐시) → 온라인 복구 시 자동 동기화

⚠️ 스코어 입력 중 앱 종료 시 로컬 임시저장, 재진입 시 복원 팝업 노출

### **B-5. 스코어보드**

* 플레이어별 합계 / 전반 / 후반

* 홀별 비교 테이블

* 입력 진행률 표시 ("홍길동: 12/18홀 입력")

* 플레이어 이름 탭 → 해당 플레이어 홀별 기록 상세

* 실시간 구독 (Firestore onSnapshot)

### **B-6. 라운드 종료 / 결과**

* 종료 확인 (HOST 전용)

  * 미입력 홀 목록 표시 \+ 경고

  * "종료 후 X시간 내 수정 가능" 정책 안내

* 결과 요약

  * 순위 / 총타 / 전반 후반 분리

  * 이미지 / 링크 공유 (선택, 2차)

## **3-5. 코스 탭 (C)**

* 코스 목록 / 검색 (지역 / 이름)

* 즐겨찾기 / 최근 사용 코스 그룹

* 코스 상세

  * 티셋 선택기

  * 홀별 Par / 거리 리스트

  * 코스 메모 / 지도 / 사진 (선택, 2차)

## **3-6. 내 기록 탭 (D)**

* 내 라운드 기록 (기간 / 코스별 / 월별 필터)

* 통계 (선택, 2차): 평균 타수, 최근 추이, 베스트 라운드, 코스별 성적

* 프로필 / 설정

  * 닉네임 / 핸디캡 / 기본 티

  * 알림 설정

  * 로그아웃 / 회원 탈퇴 (탈퇴 시 데이터 처리 정책 명시)

## **3-7. 보조 화면 / 팝업 (E)**

* 초대 관리 (Host): 링크/코드 재생성, 참가자 강퇴/역할 변경

* 에러 / 빈 상태: 코스 없음 / 네트워크 오류 / 초대 만료 등

# **6\. Firestore 데이터 모델 (보완)**

## **6-1. 컬렉션 구조 요약**

| 컬렉션 경로 | 주요 필드 | 비고 |
| :---- | :---- | :---- |
| courses/{courseId} | name, region, holesCount, status, version, updatedAt | 운영자 관리 |
| courses/{courseId}/teesets/{teeId} | name, gender, rating, slope |  |
| courses/{courseId}/holes/{holeNo} | par, handicapIndex, order, distances: {teeId: number} | 홀 번호 '1'\~'18' 권장 |
| rounds/{roundId} | hostUid, courseId(denorm), teeId(denorm), holes, scheduledAt, status, inviteCode, createdAt, updatedAt | DRAFT→IN\_PROGRESS→FINISHED |
| rounds/{roundId}/participants/{uid} | uid, nickname(denorm), role, joinStatus, holesEntered, totalOut, totalIn, total, updatedAt | 보드 즉시 표시용 집계 |
| rounds/{roundId}/scores/{uid} | holes: {'1': {strokes, putts, fairway, ob, hazard, bunker}, ...}, updatedAt | 유저당 1문서 |
| invites/{code} | roundId, createdBy, expiresAt, maxUses, useCount, active | 문서ID \= 초대 코드 |

## **6-2. scores 서브컬렉션 스키마 상세**

유저당 1개 문서로 18홀 데이터를 맵 구조로 저장합니다. 홀별 세부 기록(퍼트, Fairway, OB 등)을 위해 단순 숫자 대신 객체 구조로 확장합니다.

| 필드 | 타입 | 설명 | 필수 |
| :---- | :---- | :---- | :---- |
| holes | Map\<string, HoleData\> | 키: 홀 번호('1'\~'18'), 값: HoleData 객체 | Y |
| holes.{n}.strokes | number | 타수 | Y |
| holes.{n}.putts | number | 퍼트수 | N |
| holes.{n}.fairway | boolean | null | Fairway 안착 여부 | N |
| holes.{n}.ob | number | OB 개수 | N |
| holes.{n}.hazard | boolean | 헤저드 여부 | N |
| holes.{n}.bunker | boolean | 벙커 여부 | N |
| updatedAt | Timestamp | 마지막 업데이트 시각 | Y |

⚠️ 스코어를 홀마다 별도 문서로 저장하면 읽기/쓰기 비용이 급증합니다. 반드시 유저당 1문서 구조를 유지하세요.

## **6-3. 상태 머신**

### **라운드 상태**

| 상태 | 진입 조건 | 허용 액션 | 다음 상태 |
| :---- | :---- | :---- | :---- |
| DRAFT | 라운드 생성 직후 | 설정 수정, 참가자 초대 | IN\_PROGRESS |
| IN\_PROGRESS | HOST가 "시작" 또는 첫 스코어 입력 | 스코어 입력, 보드 조회, 종료 | FINISHED |
| FINISHED | HOST가 종료 확인 | 결과 조회, (제한적) 수정 | \- |

## **6-4. 집계 전략**

* 초기 (v1): 클라이언트에서 totalOut / totalIn / total 계산 후 participants 문서에 쓰기

* 권장 (v1.5+): Cloud Functions onWrite 트리거로 서버 집계 — 클라이언트 조작 방지

* 스코어보드 조회: participants 구독만으로 즉시 표시, scores는 상세 보기 시에만 읽기 (비용 절감)

# **7\. Firebase 보안 규칙**

## **7-1. Firestore Rules 원칙**

| 컬렉션 | 읽기 권한 | 쓰기 권한 |
| :---- | :---- | :---- |
| courses/\*\* | 모든 로그인 사용자 | admin 커스텀 클레임 보유자만 |
| rounds/{roundId} | 참가자만 (participants 확인) | HOST만 (status 변경 등) |
| rounds/{roundId}/participants/\*\* | 참가자만 | 본인 문서: 본인 / 관리 액션: HOST |
| rounds/{roundId}/scores/{uid} | 참가자만 | 본인 uid 일치 \+ FINISHED 아닌 경우 |
| invites/{code} | 모든 로그인 사용자 | Functions만 (직접 쓰기 차단) |

## **7-2. 추가 보안 고려사항**

* FINISHED 상태 라운드: 스코어 쓰기 차단 (또는 종료 후 2시간 내 허용)

* invites 코드 생성: Cloud Functions에서만 수행 (클라이언트 직접 생성 차단)

* HOST 교체: 탈퇴/강제 이탈 시 처리 정책 필요 (2차에서 정의)

* rate limiting: Cloud Functions에서 초대 코드 생성 횟수 제한 (스팸 방지)

# **8\. 에러 처리 및 엣지 케이스**

## **8-1. 에러 시나리오 목록**

| 시나리오 | 발생 시점 | 처리 방법 |
| :---- | :---- | :---- |
| 네트워크 오류 | 전체 화면 | 토스트 \+ 오프라인 캐시 활성화, 자동 재시도 |
| 초대 코드 만료 | 코드 입력 / 딥링크 진입 | "초대가 만료되었습니다" 안내 \+ 호스트에게 재요청 안내 |
| 초대 코드 최대 사용 초과 | 코드 입력 | "참가 인원이 가득 찼습니다" 안내 |
| FINISHED 라운드 참가 시도 | 코드 입력 | "이미 종료된 라운드입니다" 안내 |
| 중복 참가 | 코드 입력 / 딥링크 | 바로 라운드 상세로 이동 |
| 코스 데이터 없음 | 라운드 생성 Step 1 | 빈 상태 UI \+ 코스 요청 CTA |
| 미입력 홀 존재 시 종료 시도 | 라운드 종료 | 미입력 홀 목록 모달 \+ 강제 종료 여부 확인 |
| 앱 강제 종료 후 재진입 | 스코어 입력 중 | 로컬 임시저장 복원 팝업 |
| 동시 업데이트 충돌 | 스코어 입력 (서버) | Firestore 트랜잭션으로 처리, 충돌 시 재시도 |
