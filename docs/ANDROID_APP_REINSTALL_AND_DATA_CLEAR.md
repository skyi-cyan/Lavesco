# Android 앱 재설치 / 앱 데이터 삭제 방법

Firebase 인증(UNAUTHENTICATED) 문제 후 SHA-1 확인·등록이 끝났다면, 아래 중 한 가지 방법으로 앱을 새로 띄워 보세요.

---

## 방법 1: 앱 다시 빌드 후 기기에 설치

### 1-1. 클린 빌드 (선택)

터미널에서:

```powershell
cd D:\NewProduct\Lavesco\mobile-rn\android
.\gradlew.bat clean
```

### 1-2. 앱 실행(빌드 + 설치)

**방법 A – React Native CLI로 실행 (권장)**

```powershell
cd D:\NewProduct\Lavesco\mobile-rn
npx react-native run-android
```

- USB로 연결된 기기 또는 실행 중인 에뮬레이터에 **자동으로 빌드·설치**됩니다.
- 이미 앱이 설치돼 있으면 **덮어쓰기 설치**됩니다.

**방법 B – Android Studio에서 실행**

1. Android Studio에서 `mobile-rn/android` 폴더를 프로젝트로 연다.
2. 상단 **Run** (▶) 버튼 클릭하거나 **Shift+F10**.
3. 연결된 기기 또는 에뮬레이터를 선택해 설치·실행.

### 1-3. 정리

- **같은 기기에 그대로 설치** → 기존 앱이 새 빌드로 교체됩니다.
- **완전히 새로 깔고 싶다면** → 아래 “방법 2”처럼 먼저 앱 삭제 후, 다시 위 1-2를 실행하면 됩니다.

---

## 방법 2: 기기에서 앱 삭제 후 재설치

### 2-1. 앱 삭제(제거)

1. 기기 **설정** 앱 실행
2. **앱** (또는 **애플리케이션** / **앱 관리**) 메뉴로 이동
3. **Lavesco** 앱 찾아서 탭
4. **삭제** (또는 **제거** / **언인스톨**) 선택 후 확인

(또는 앱 서랍에서 Lavesco 아이콘 길게 누르기 → **앱 정보** → **삭제**)

### 2-2. 다시 설치

- **방법 1**의 **1-2**처럼 `npx react-native run-android` 또는 Android Studio **Run**으로 다시 빌드·설치합니다.

---

## 방법 3: 앱 데이터만 삭제 (재설치 없이)

앱은 그대로 두고, **저장된 로그인·캐시만 지우고** 다시 로그인하는 방법입니다.

### 3-1. 앱 데이터 삭제

1. 기기 **설정** 앱 실행
2. **앱** (또는 **애플리케이션** / **앱 관리**) 메뉴로 이동
3. **Lavesco** 앱 찾아서 탭
4. **저장공간** (또는 **저장용량** / **스토리지**) 탭 선택
5. **데이터 삭제** (또는 **저장된 데이터 지우기**) 탭 후 확인

(제조사·OS 버전에 따라 **저장공간** 대신 **데이터 및 캐시** 등 비슷한 메뉴일 수 있습니다.)

### 3-2. 앱 다시 실행

- 앱 서랍에서 **Lavesco** 실행
- 로그인 화면이 나오면 **다시 로그인**
- 그 다음 **프로필 저장** 등 동작 확인

---

## 어떤 방법을 쓸지

| 상황 | 추천 |
|------|------|
| SHA-1 새로 등록했거나 google-services.json 갱신함 | **방법 1** (다시 빌드 후 설치) 또는 **방법 2** (삭제 후 재설치) |
| 앱은 그대로 두고 로그인/캐시만 초기화하고 싶음 | **방법 3** (앱 데이터 삭제) |

SHA-1/설정 변경 후에는 **방법 1 또는 2**로 한 번 새로 설치해 보는 것이 가장 확실합니다.
