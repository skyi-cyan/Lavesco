# 홀 스코어 사운드 설정 가이드 (내가 할 일)

스코어 저장 시 "버디", "파" 같은 **안내 소리(음성/효과음)**를 쓰려면 아래 순서대로 진행하면 됩니다.

---

## 진동만 되고 소리가 안 날 때

- **원인:** `res/raw` 폴더에 재생할 MP3 파일이 하나도 없기 때문입니다.
- **가장 빠른 확인 방법:** 아래 **5. Android에서 내가 할 일**대로 `mobile-rn/android/app/src/main/res/raw` 폴더를 연 뒤, **아무 짧은 MP3 파일 하나**를 **이름을 `beep.mp3`로 해서** 그 폴더에 넣고 앱을 다시 빌드해 보세요.  
  → 스코어 저장 시 해당 스코어용 파일(예: par.mp3)이 없으면 **beep.mp3**가 대신 재생됩니다.  
  → beep.mp3만 있어도 저장할 때마다 비프음이 나오므로, "사운드가 동작하는지" 먼저 확인할 수 있습니다.
- **스코어별 음성**을 쓰려면, 아래 2번처럼 각 경우에 맞는 MP3(par.mp3, birdie.mp3 등)를 같은 `res/raw` 폴더에 추가하면 됩니다.

---

## 1. MP3가 뭔가요?

- **MP3** = 소리(음성, 효과음)를 담은 **음악/오디오 파일** 형식입니다.
- 휴대폰에서 노래나 음성메모 재생할 때 쓰는 그 파일이에요.
- 우리 앱에서는 "버디", "파" 같은 **짧은 한 마디 음성** 또는 **띵동 같은 효과음**을 MP3 파일로 넣어 두면, 스코어 저장할 때 그 소리가 재생됩니다.

---

## 2. 어떤 소리를 준비해야 하나요?

아래 **14가지 경우**마다 **딱 하나씩** MP3 파일이 필요합니다.  
(예: "버디"라고 말하는 음성 1개 = `birdie.mp3` 한 파일)

### 2-1. 기본 스코어 (7개)

| 번호 | 파일 이름 (정확히 이렇게) | 재생되는 상황 | 넣을 소리 예시 |
|------|---------------------------|---------------|----------------|
| 1 | eagle.mp3 | 이글 쳤을 때 | "이글" 음성 또는 짧은 축하음 |
| 2 | birdie.mp3 | 버디 쳤을 때 | "버디" 음성 |
| 3 | par.mp3 | 파 쳤을 때 | "파" 음성 |
| 4 | bogey.mp3 | 보기 쳤을 때 | "보기" 음성 |
| 5 | double_bogey.mp3 | 더블보기 | "더블보기" 음성 |
| 6 | triple_bogey.mp3 | 트리플보기 | "트리플보기" 음성 |
| 7 | yangpa.mp3 | 양파(더블파 이상) | "양파" 음성 |

### 2-2. 연속 파 특수 사운드 (4개)

| 번호 | 파일 이름 | 재생되는 상황 | 넣을 소리 예시 |
|------|-----------|---------------|----------------|
| 8 | par_3_streak.mp3 | 3홀 연속 파 | "3연속 파 세이브" 음성 |
| 9 | par_4_audi.mp3 | 4홀 연속 파 | "아우디 파" 음성 |
| 10 | par_5_olympic.mp3 | 5홀 연속 파 | "올림픽 파" 음성 |
| 11 | par_6_skilled.mp3 | 6홀 연속 파 | "실력파 파" 음성 |

### 2-3. 버디 특수 사운드 (2개)

| 번호 | 파일 이름 | 재생되는 상황 | 넣을 소리 예시 |
|------|-----------|---------------|----------------|
| 12 | birdie_birdie.mp3 | 연속으로 버디 2개 이상 | "버디 버디" 음성 |
| 13 | cycle_birdie.mp3 | 파3→파4→파5 연속 홀에서 각각 버디 | "사이클 버디" 음성 |

**정리:** 위 13개 파일 이름을 **그대로** 쓰고, 각각 해당 문구를 말하는 **짧은 음성**을 MP3로 녹음하거나 만들어서 넣으면 됩니다.  
(전부 다 만들 필요는 없고, 필요한 것만 만들어도 됩니다. 없는 파일은 해당 시에는 **기본 비프음(beep.mp3)**이 재생되고, beep.mp3도 없으면 진동만 됩니다.)

## 3. MP3 파일은 어떻게 만들 수 있나요?

- **직접 녹음:** 스마트폰 음성메모 앱으로 "버디", "파" 등 말하고, PC로 옮긴 뒤 MP3로 변환
- **TTS(음성 합성):** 네이버 클로바, Google TTS 등으로 문장 입력 후 음성 파일로 저장
- **제작 의뢰:** 음성/효과음 제작 업체에 위 문구 리스트 주고 MP3 제작 요청
- **무료 효과음:** 무료 효과음 사이트에서 짧은 알림음 다운로드 후, 파일 이름만 위 표대로 바꿔서 사용 (문구 대신 효과음만 쓸 경우)

---

## 4. 파일 이름 규칙 (꼭 지켜야 함)

- **반드시 소문자만** (eagle, par, birdie_birdie)
- **공백 없음** → 대신 **언더스코어(_)** 사용 (예: `double_bogey.mp3`, `par_3_streak.mp3`)
- **확장자는 .mp3** (예: `birdie.mp3`)

## 5. Android에서 내가 할 일 (단계별)

1. **폴더 열기**  
   PC에서 아래 폴더를 엽니다.  
   ```
   Lavesco 폴더
   └── mobile-rn
       └── android
           └── app
               └── src
                   └── main
                       └── res
                           └── raw
   ```  
   즉, **`mobile-rn/android/app/src/main/res/raw`** 폴더입니다.  
   (처음에는 `raw` 폴더가 없을 수 있어요. 그럼 **`raw` 폴더를 새로 만든다**고 생각하면 됩니다.)  
   **주의:** 이 폴더에는 **MP3 파일만** 넣어야 합니다. README나 다른 문서를 넣으면 Android 빌드가 실패합니다.

2. **MP3 파일 넣기**  
   만든(또는 준비한) MP3 파일들을 **이름을 위 표와 똑같이** 바꾼 뒤, 이 **`raw` 폴더 안에** 그대로 복사해 넣습니다.  
   - 예: `birdie.mp3`, `par.mp3`, `eagle.mp3`, `par_3_streak.mp3` …  
   - **소리 테스트만 할 때:** 아무 짧은 MP3 하나를 **`beep.mp3`** 로 이름 지어서 넣으면, 스코어별 파일이 없을 때 이 비프음이 재생됩니다.  
   - **주의:** `raw` 안에는 **파일만** 넣고, 그 안에 다시 폴더를 만든다든지 하면 안 됩니다.

3. **앱 다시 빌드**  
   터미널에서:  
   ```bash
   cd mobile-rn
   npx react-native run-android
   ```  
   빌드가 끝나면, 스코어 저장할 때 해당하는 MP3가 재생됩니다.

## 6. iOS에서 내가 할 일 (단계별)

1. **Xcode로 프로젝트 열기**  
   `mobile-rn/ios/MobileRn.xcworkspace` (또는 `.xcodeproj`)를 더블클릭해서 Xcode를 엽니다.

2. **MP3 파일을 프로젝트로 가져오기**  
   - 왼쪽 프로젝트 탐색기에서 프로젝트 이름(예: MobileRn) 클릭  
   - 메뉴 **File → Add Files to "MobileRn"…** 선택  
   - 준비한 MP3 파일들(eagle.mp3, birdie.mp3, par.mp3 등)을 **한 번에 선택**  
   - **"Copy items if needed"** 체크  
   - **"Add to targets: MobileRn"** 체크 후 추가

3. **앱 번들에 포함시키기**  
   - 상단에서 **타겟(Target)**이 **MobileRn**인지 확인  
   - **Build Phases** 탭 클릭  
   - **Copy Bundle Resources** 섹션 펼치기  
   - **+** 버튼 눌러서 방금 추가한 MP3 파일들을 **전부** 여기에 넣기  
   - (이미 목록에 있으면 그대로 두면 됩니다.)

4. **다시 빌드 후 실행**  
   Xcode에서 **Run(▶)** 버튼으로 빌드하고 실행하거나, 터미널에서:  
   ```bash
   cd mobile-rn
   npx react-native run-ios
   ```

## 8. 파일을 아직 안 넣으면 어떻게 되나요?

- **앱은 그대로 동작합니다.**  
- 스코어 저장도 되고, 화면도 정상입니다.  
- 다만 **해당하는 경우에 소리만 나지 않습니다.**  
- 나중에 MP3를 넣고 다시 빌드하면 그때부터 소리가 납니다.

궁금한 점 있으면 이 문서를 기준으로 "몇 번 단계에서 막히는지" 알려주시면 됩니다.
